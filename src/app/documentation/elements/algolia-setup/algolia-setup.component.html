Live Search Set Up

Lara-Commerce has two options for performing searches a Basic approach or using Algolia. The basic approach uses a mysql query to on the products table to return search results. The algolia approach is a bit more complex and some of the advantages of using algolia are listed below. For more information please visit the algolia website

Advantages

  • Customise Search Results
  • Spell Checking
  • Faster Searches

Disadvantage

  • Cost Money

The basic approach is set up by default, so no action is needed to set it up. In order to set up the algolia approach please follow the guide below

Step 1: Set Up an Algolia account

In order to set up an algolia account go to there website and click create an account. Following the instructions and sign in to your new account.

Step 2: Set Up API Keys

Once you have created an account visit your algolia dashboard and on your side bar click the API Key link. The API Key page will contain information about your Application ID and your Admin API Key. You will need to include these in your .env file.

ALGOLIA_APP_ID= //your Application ID
ALGOLIA_SECRET= //you AdminAPI Key

Step 3: Set Up Search Products

In the Lara-Commerce app/Product.php file look for the search method and comment it out. 

    
      /**
       * performs a search if algolia is not used
       * comment out if algolia is used
       *
       * @param $query
       * @return mixed
       */
      //public function search($query)
      //{
      //    return $this->where('title', 'like', "%$query%");
      //}

Next at the top of the app/Product.php look for the trait Searchable and uncomment it. Algolia is now your primary search engine. 

    
      class Product extends Model
      {
          use Searchable;
