Set Up Laravel's Dusk

Lara-Commerce has an been tested using Laravel's Dusk. However, to run the Laravel's Dusk test successfully you must first create a test database. If you have not created testing database, then please following the instructions at Setting up PHPUnit

Step 1: Point Laravel's Dusk to the Test Database

In order to point Laravel's Dusk to the testing database simple generate a new .env.example file using the php artisan publish:env command. 

  php artisan publish:env

After the .env.example file has been generated, it must be renamed .env.dusk.local . The .env.dusk.local file will provide all the settings that will be used for testing the browser, including the test database. Open the .env.dusk.local file and delete everything but the following settings. Make sure that the DB_CONNECTION and DB_DATABASE reflect your testing database.

APP_ENV=local
APP_KEY= //app key from your original .env file
APP_DEBUG=true
APP_LOG_LEVEL=debug
APP_URL= //host name from homestead

DB_CONNECTION=sqlite_testing

You are now ready to begin testing the browser. To run browser tests simply enter php artisan dusk in your terminal. 

  php artisan dusk
